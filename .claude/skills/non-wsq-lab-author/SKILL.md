# Non-WSQ Lab Author

1. Map each published topic to one or more observable build outcomes.
2. Prefer one capstone that grows across labs; state prerequisites and preserve checkpoints.
3. For each lab write: Goal, What you will build, Prerequisites, numbered Steps, Test it, Troubleshooting, Evidence and Reflection.
4. Make every step executable: name the file, agent request, review action or verification command.
5. Require learners to inspect plans and diffs before accepting generated code.
6. Use synthetic data and secret placeholders. Never instruct learners to paste credentials into prompts.
7. Keep tooling names prefixed `non-wsq-` and exclude funding, SSG, TRAQOM and formal assessment content.
